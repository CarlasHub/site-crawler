Cat Crawler · Technical overview · Drag-to-install bookmarklet

Website validation crawler with an in-page bookmarklet runner.

Cat Crawler crawls internal navigation, follows redirects, tests parameter variants, detects soft failures, and groups URL-pattern inconsistencies. Drag the bookmarklet to your bookmarks bar to open the crawler against the page you are currently viewing.

Discovery model

Starts from sitemap.xml when available, then traverses internal a[href] links and form[action] routes while respecting robots.txt.

Validation checks

Classifies navigation into broken URLs, redirect issues, parameter-handling issues, soft failures, and prioritised impact levels.

Runtime controls

Supports host scoping, path exclusions, language-agnostic path limits, job-page suppression, concurrency control, and optional parameter audit.

Output model

Returns referrer-aware audit entries plus dedicated sections for redirect chains, soft failures, URL patterns, impact analysis, and TXT/CSV export.

Run model

1

Seed the crawl

Enter the homepage or scoped start path, for example https://example.com or https://example.com/en.

2

Define crawl rules

Add excluded paths and path limits such as /jobs, /careers, or /job max 5.

3

Choose validators

Enable HTTP status checks and parameter audit when the target site uses query-driven routing or filtered search pages.

4

Execute the run

The crawler tests each discovered navigation entry, records referrers, follows redirects fully, and classifies the result.

5

Review the report

Use the validation report, redirect audit, soft-failure audit, pattern audit, and impact analysis before exporting TXT or CSV.

Recommendation: turn on Broken link quick check for production audits and Parameter audit for search/results routes.

Bookmarklet setup (Cat Crawler)

The bookmarklet injects a floating iframe pointed at the deployed app and passes the current page URL as the seed.

  1. Drag the Drag Cat Crawler 😼 button above to your bookmarks bar.
  2. Open any page you want to inspect.
  3. Click the saved bookmark to load the latest bookmarklet.js from GitHub Pages.
  4. Drag the header to move the panel and resize it from any of the four corners.

Audit scope checklist

Use this before exporting or sharing results:

  • Confirm sitemap and robots.txt were reachable
  • Check whether excluded sections and path limits hid expected routes
  • Review redirect chains for loops, multiple hops, and dropped parameters
  • Review soft failures on 200 responses before treating them as valid
  • Use impact analysis to prioritise repeated or core-flow issues

Bookmarklet update model
