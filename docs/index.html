Cat Crawler 😼 How to use

What it does

Discovers internal URLs from a homepage using sitemap + page links while respecting robots.txt.

Why it matters

Keep crawls clean by excluding noisy sections and capping job pages so content QA stays focused.

What’s new

Bookmarklet mode (“Cat Crawler”), faster queueing, and cleaner exclusion matching.

Built-in outputs

Export TXT/CSV with final URLs, redirects, and optional status codes.

How to use the crawler

1

Enter a homepage URL

Use the root page you want to crawl, e.g. https://example.com.

2

Add exclusions

One path per line, like /jobs, /careers, /admin.

3

Set path limits

Cap busy sections, e.g. /job max 5 pages.

4

Choose options

Adjust max pages/concurrency; toggle “Ignore job pages” or “Broken link quick check.”

5

Run and export

Click “Run crawl” and download TXT/CSV from Results.

Tip: enable “Broken link quick check” to capture HTTP status codes and spot 404s quickly.

Bookmarklet setup (Cat Crawler)

Run the crawler inside any page you’re visiting, using a floating in-page panel.

  1. Open docs/bookmarklet.js and set APP_ORIGIN to your deployed app URL.
  2. Create a new browser bookmark and paste the file contents as the URL (prefix with javascript:).
  3. Click the bookmark on any site to open Cat Crawler with the current URL prefilled.

Quality checklist

Use this to keep runs consistent across teams:

  • Confirm sitemap available and robots.txt respected
  • Exclude careers/job sections unless needed
  • Cap pagination-heavy paths
  • Export CSV and share with stakeholders

Results you can trust

Each URL includes original + final destination, optional HTTP status, and flags for duplicates.

Team-friendly presets

Save configurations per client, export to JSON, and re-use for future audits.
